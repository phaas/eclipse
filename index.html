Keyboard Shortcuts & other Nice Things

Let's start..

  • Create a new project (Ctrl + N)
  • Type java into the filter
  • Select Java Project
  • Give it a name...
  • Finish

Create a class

  • New File (Ctrl+N)
  • Type class and hit Enter (once or twice)
  • Give it a name.. say "Main"
  • Optionally: pick a package.
    • Use the accessibility shortcuts to select the field: alt + k
    • Let's call it runner
  • Enter to close the dialog

Generate a main method

  • Position the cursor anywhere whithin the class
  • Type main
  • Press Ctrl+Space to get suggestions
  • Choose the main method template
    • Use the arrow keys to move between suggestions

Use basic content suggest (auto complete)

  • Press Ctrl + Space anywhere to get a list of variables, methods and other objects in your current scope
  • Eclipse recognizes upper-case letters as you type and matches it to variables, methods, classes, ...
public static void main(String[] args) {
		String firstPart = "Hello";
		String secondPart = "World";
}
  • Type System.out.println(fP and press Ctrl + Space
  • fP completes as firstPart
  • sP completes as secondPart
  • AIOOBE completes as ArrayIndexOutOfBoundsException (really!)

Opening Types and Resources

  • Eclipse knows two basic objects:
    • Types - Basically Java classes, somewhere in your project, or in a .jar file on your project's build path. Find these with Ctrl + Shift + T
    • Resources - Files within your project. This does not look insideof archives/jars. Find these with Ctrl + Shift + R

    • Examples

    • A java class from the SDK: Ctrl+Shift+T and typing FiInStr matches the class FileInputStream
    • A file: Ctrl+Shift+R and typing *.java matches Main.java
    Eclipse lets you mix upper & lower case letters. You can only type the upper-case letters in the type that you're looking for or add lower-case letters as needed to limit the number of suggestions

Launching your application (or test)

  • Right-click in the editor, select Run As then Java Application. This works if the file in the editor has a main method.
  • Without using the mouse: Press Alt + Shift + X to bring up the launch menu, then type x to choose Java Application.
    • No need to wait for the launch menu to show up.. Type as fast as you want!
  • This also works for launching JUnit Tests: Alt + Shift + X, T
  • Want to launch the app / test in Debug mode? Use Alt + Shift + D (Debug) instead of Alt + Shift + X (Execute)

Relaunching an application (or test)

  • Need to rerun the last program or test?
    • Ctrl + F11 to Run
    • F11 to Debug
  • WARNING: By default, Eclipse uses a very annoying setting that tries to guess what you're trying to run! Let's fix this so that Eclipse always relaunches the last program that you ran, no matter what you're looking at in the editor now.
    • Window -> Preferences -> Run/Debug -> Launching
  • Good news: This also fixes the behavior of the Launch and Debug buttons in the toolbar!

Finding any action in Eclipse

  • Using the menu to get to the preferences isn't fun.. but there's a shortcut!
  • Ctrl + 3 = Find Anything!
  • Type the name of any action, dialog, view, perspective, preference, ...
  • For example: Press Ctrl + 3 and type Launching to find the preferences pane used in the previous slide.
  • Type breakpoints to open the breakpoints view
  • Type debug to find the Debug perspective
