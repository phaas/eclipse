How to get the most out of Eclipse's shortcuts and source manipulation tools

  1. Learn about Eclipse's capabilities (have a look at all of the slides)
  2. When writing code, try to think in terms of the source and refactoring primitives offered by Eclipse
  3. Let Eclipse write some of the code for you
  4. Pay attention to the keyboard shortcuts of the actions that you find most useful

Let's start..

  • Create a new project (Ctrl + N)
  • Type java into the filter
  • Select Java Project
  • Give it a name...
  • Finish

Create a class

  • New File (Ctrl + N)
  • Type class and hit Enter (once or twice)
  • Give it a name.. say "Main"
  • Optionally: pick a package.
    • Use the accessibility shortcuts to select the field: Alt + K
  • Press Enter to close the dialog

Generate a main method

  • Position the cursor anywhere within the class
  • Type main
  • Press Ctrl + Space to get suggestions
  • Choose the main method template
    • Use the arrow keys to move between suggestions

Basic content suggest (auto complete)

  • Press Ctrl + Space anywhere to get a list of variables, methods and other objects in your current scope
  • Eclipse recognizes upper-case letters as you type and matches it to variables, methods, classes, ...
public static void main(String[] args) {
		String firstPart = "Hello";
		String secondPart = "World";
}
  • Type System.out.println(fP and press Ctrl + Space
  • fP completes as firstPart
  • sP completes as secondPart
  • AIOOBE completes as ArrayIndexOutOfBoundsException (really!)

Word Completion

  • Sometimes Eclipse can complete words even when there is not enough information about the context.
  • This can be handy when editing text files (especially html, xml) or when Eclipse can't figure out what you're looking for otherwise.
  • Type at least one character and press Alt + /. Press it again to get the next suggestion
  • Eclipse will find words with the same prefix in other opened Editors.

Opening Types and Resources

Eclipse knows two basic objects

  • Types - Basically Java classes, somewhere in your project, or in a .jar file on your project's build path. Find these with Ctrl + Shift + T
  • Resources - Files within your project. This does not look insideof archives/jars. Find these with Ctrl + Shift + R

Examples

  • A java class from the SDK: Ctrl + Shift + T and typing FiInStr matches the class FileInputStream
  • A file: Ctrl + Shift + R and typing *.java matches Main.java
Eclipse lets you mix upper & lower case letters. You can only type the upper-case letters in the type that you're looking for or add lower-case letters as needed to limit the number of suggestions

Open Declaration

  • The Open Declaration navigates to the selected Class, variable declaration or method definition
  • There are many ways of triggering this action:
    • Press F3
    • Hold down Ctrl and left click on an element
    • Right-click on an element and select Open Declaration from the context menu

Quick Outline

  • Press Ctrl + O to view a quick outline of the class opened in the editor
  • While in the quick outline, type anything to filter the displayed items

Quick Type Hierarchy

  • When working with Interfaces the Open Declaration action will go to the Interface definition.
  • To quickly find the implementation(s) of an Interface method, press Ctrl + T

Launching your application (or test)

  • Right-click in the editor, select Run As then Java Application. This works if the file in the editor has a main method.
  • Without using the mouse: Press Alt + Shift + X to bring up the launch menu, then type x to choose Java Application.
    • No need to wait for the launch menu to show up.. Type as fast as you want!
  • This also works for launching JUnit Tests: Alt + Shift + X, T
  • Want to launch the app / test in Debug mode? Use Alt + Shift + D (Debug) instead of Alt + Shift + X (Execute)

Relaunching an application (or test)

  • Need to rerun the last program or test?
    • Ctrl + F11 to Run
    • F11 to Debug
  • WARNING: By default, Eclipse uses a very annoying setting that tries to guess what you're trying to run! Let's fix this so that Eclipse always relaunches the last program that you ran, no matter what you're looking at in the editor now.
    • Window -> Preferences -> Run/Debug -> Launching
  • Good news: This also fixes the behavior of the Launch and Debug buttons in the toolbar!

Finding any action in Eclipse

  • Using the menu to get to the preferences isn't fun.. but there's a shortcut!
  • Ctrl + 3 = Find Anything!
  • Type the name of any action, dialog, view, perspective, preference, ...
  • For example: Press Ctrl + 3 and type Launching to find the preferences pane used in the previous slide.
  • Type breakpoints to open the breakpoints view
  • Type debug to find the Debug perspective

Structural Select

  • Regular select: Hold Shift and move the cursor Left,Right, ...
  • Block / Structural select: Hold Alt + Shift and move the cursor UpLeftRightor Down
  • The exact selection depends on what your cursor is focused on..

Quick Fix: Automatically modify source code

  • Eclipse has great support for automatically resolving compiler errors. Use this to your advantage!
  • Write code the way you want your classes to work and let Eclipse make it so.
  • Quick fixes can be selected via many "error" markers or by hovering over an error
  • Much faster: Press Ctrl + 1. Use this always!

Quick Fix: Create a Class

  • Type the name of a class that you'd like to create: new MyClass();
  • Press Ctrl + 1
  • Choose Create Class
  • Change the package, superclass, implemented interfaces, ... as needed.
package x.y.z;

public class MyClass {

}

Quick Fix: Assign to Local Variable

  • Select any value expression (New class, non-void method, literal, ...) and press Ctrl + 1
  • Select Assign Statement to new local variable or Assign Statement to new field
  • Modify the variable name (or choose another suggestion with the up/down arrows) and press Enter to complete the action
new MyClass();
MyClass myClass = new MyClass();
  • Alternative shortcut: Ctrl + 2 followed by L (local variable) or F (field)

Quick Fix: Correct type

  • When assigning variables, Eclipse can correct the "left hand side" of the assignment based on the type of the "right hand side"
  • On an invalid assignment or return statement, press Ctrl + 1 and choose Change type of ...
void fixTypes() {
    String s = 3 + 4;
    return s;
}
int fixTypes() {
    int s = 3 + 4;
    return s;
}

Quick Fix: Generate a method

  • Write method signature of a method that you'd like to create and press Ctrl + 1
  • Select Create method ... from the Quick Fix menu and press Enter
  • Pres Tab to cycle through the method signature parts and adjust them as needed
MyClass mc = new MyClass();
    mc.myFunction(1, "string", new Date());

public class MyClass {
    public void myFunction(int i, String string, Date date) {
        // TODO Auto-generated method stub
    }
}

Quick Fix: Add or Remove a parameter of a method or constructor

  • Add or remove parameters when calling an existing method, then press Ctrl + 1
  • Choose Change method ...: Remove Parameter or Change method ...: Add Parameter and press Enter
  • When adding a parameter, edit the new parameter type and name
  • When removing a parameter, fix any compiler errors

More source code manipulation: Source and Refactor tools

  • Eclipse has a wide range of sophisticated source code manipulation tools
  • Right click in the editor and choose the Source or Refactor submenus
  • Faster: Alt + Shift + S (source) and Alt + Shift + T (refactor)

Source: Format

  • Found in the Source menu or by pressing Ctrl + Shift + F
  • "Fixes" the formatting, indentation, line breaks in the current file or selected region
  • Best to always or never use this on a whole file. Depending on the formatting rules that are loaded and the length of the lines, things can look a little.. odd
  • Most useful when selecting specific sections (use the structural select mode)

Source: Add Import

  • Found in the Source menu or by pressing Ctrl + Shift + M
  • Use this fully qualified class name or a static method
  • Eclipse will add an import statement for the class or a import static statement for the method
java.util.Arrays.asList("Import","This");
import java.util.Arrays;
    ...
    Arrays.asList("Import","This");
import static java.util.Arrays.asList;
    ...
    asList("Import","This");

Source: Organize Imports

  • Found in the Source menu or by pressing Ctrl + Shift + O
  • Organizes the import statements:
    • Remove any unused imports
    • Add missing imports. Eclipse will prompt you if multiple classes with the same name are found
    • Replace any .* imports with the classes actually used (up to a certain limit)
    • Group all imports by the top level package and sort the import statements alphabetically
  • Use this all the time!
  • Eclipse can be configured to organize imports automatically on save:
    • Find the Java / Editor / Save Actions preference pane (Ctrl + 3)

Source: Generate Getters and Setters

  • Found in the Source menu or by pressing Alt + Shift + S then R
  • Select the fields and accessor types desired (or Alt + A for all)
  • Modify the other options as needed (Tip: Set the insertion point to last member)
public class MyClass {
    private String a;
    private boolean d;
}
public String getA() {
        return a;
    }
    public void setA(String a) {
        this.a = a;
    }
    public boolean isD() {
        return d;
    }
    public void setD(boolean d) {
        this.d = d;
    }

Source: Generate Delegate Methods

  • Useful when writing a class that implements an Interface but wants to delegate to an (existing) implementation
  • Example: Your class implements java.util.List and uses a java.util.ArrayList internally.
  • Select Generate Delegate Methods... from the Source menu, pick the delegate object(s) which (missing) interface methods to delegate.
public class MyList<T> implements List<T> {
    private final List<T> delegate = new ArrayList<T>();
}
    public boolean add(T e) { return delegate.add(e); }
    public void clear() { delegate.clear(); }
    public T get(int index) {return delegate.get(index);}
This pattern should be avoided if possible, but this beats writing the delegates by hand when absolutely needed..

Source: Generate Constructor using Fields...

  • Found in the Source menu or by pressing Alt + Shift + S then O
  • Select the field(s) of the class to be initialized in the constructor
  • Optional: Change order of parameters, omit call to 'super()', or change the constructor visibility
public class MyClass {
    private String a;
    private boolean d;
}
    public MyClass(String a, boolean d) {
        super();
        this.a = a;
        this.d = d;
    }

Refactor: Rename

  • Found in the Refactor menu or by pressing Alt + Shift + R
  • Renames Classes, Fields, Methods, Packages
  • Eclipse will find all references in this (and all linked) project and rename them as well!
  • Pressing Alt + Shift + R once lets you make the change "inline" in the editor.
  • Pressing it twice opens up the Rename dialog with advanced options.
    • Don't update references
    • Rename Getter / Setter methods when renaming a field
    • Generate new Getter / Setter methods, make old ones delegates
    • Search for "text" matches (i.e. XML files) when renaming Classes

Refactor: Extract Interface

  • Found in the Refactor menu
  • Creates an Interface containing one or more methods from the current Class and lets the class implement the interface.
public class MyImpl {
    public void methodA() { }
    public int methodB(int a, int b) { return 0; }
    public void methodC(String s) { }
}
public interface MyInterface {
    void methodA();
    int methodB(int a, int b);
}

Refactor: Move

  • Found in the Refactor menu or by pressing Alt + Shift + V
  • Moves static Fields and Methods to another Class
  • Moves fields between Classes and Inner Classes
  • "Textual Move" of fields to other classes

Refactor: Extract Superclass

  • Found in the Refactor menu
  • Move selected Fields and Methods to a new Class, make the current Class class extend the new class.
  • If the current Class already has a superclass, insert the new Class between the two

Refactor: Pull Up / Pull Down

  • Found in the Refactor menu
  • Move Fields and Methods to a super or subclass
  • Eclipse checks all dependencies of the moved elements and will offer to adjust the visibility or warn of potential problem created by the move

Refactor: Inline method or declaration

  • Found in the Refactor menu or by pressing Alt + Shift + I
  • When used on a local variable, Eclipse will remote it and replace all uses with the assigned value.
    • The variable must only be assigned to once to use this refactoring
  • When used on a Method, Eclipse will insert the entire method body at the location(s) where the method is used.
    • The method can then be removed

Refactor: Extract method

  • Found in the Refactor menu or by pressing Alt + Shift + M
  • Select one or more statements that can be turned into a method
    • The statements may have any number of inputs
    • Only one variable outside of the selected line's scope may be assigned to (the generated method can only have a single return value
    • Eclipse will find similar lines elsewhere in the class and will offer to replace all occurrences

Generate a for loop

  • Type for or foreach and press Ctrl + Space
  • Choose the loop template, then fill in the required parts (press Tab to cycle through the template fields
  • Alternative way of writing a foreach loop:
    • Type for (x x : items)
    • With the focus still on items, press Ctrl + 1 and let Eclipse correct the type of 'x' (Change type of 'x' to 'String')

Find / Incremental Find

  • Eclipse has a standard Find and Replace feature: Ctrl + F
    • Regular Expression support
    • Scope: Search / Replace in the whole file, or only in the selected lines
  • Sometimes the search dialog gets in the way and the incremental find is more useful
    • Press Ctrl +J and start typing
    • Watch the bottom status bar to see what you've typed so far
    • Get immediate feedback (found, not found) in the status bar
    • Use the Up and Down arrows to search backward and forward
    • Press Esc to exit from the incremental find mode
    • Press Ctrl + J twice to run the last search again
    • The search is case-insensitive if you only use lower-case letters. Typing any upper-case letter will make it case-sensitive

Move or Delete lines

  • Ctrl + D deletes the currently selected line(s)
  • Alt + Up and Alt + Down move the selected lines up and down
  • Very simple features, but incredibly handy!

Browse and customize shortcuts

  • All of the available actions and keyboard bindings can be customized in the preferences panel
  • Preferences: General -> Keys
